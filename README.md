# Тестовое задание PHP разработчик (Laravel)

Проект реализует три основные задачи:

1. **Консольная команда** `jokes:fetch` для получения случайных шуток из публичного API (https://official-joke-api.appspot.com/random_joke) и сохранения их в БД. Планировщик запускает команду каждые 5 минут.
2. **REST API** `/api/jokes` возвращает все сохранённые шутки в формате JSON.
3. J**avaScript**  + динамические поля
4. **JavaScript счётчик посещений пропущен** (дополнительное задание) Пропущено. Нет времени.


---

## 🚀 Быстрый старт (локальный запуск)

### Требования
- PHP >= 8.4
- Composer
- Node.js & npm
- SQLite (или другая СУБД)

### Установка

1. Клонируйте репозиторий:
   ```bash
   git clone https://github.com/YugeneYak/joke-fetcher.git
   cd joke-fetcher

Установите зависимости PHP:

bash
composer install
Установите зависимости Node.js и соберите фронтенд:

bash
npm install && npm run build
Создайте файл окружения:

bash
cp .env.example .env
Настройте .env:

Укажите DB_CONNECTION=sqlite

Убедитесь, что DB_DATABASE закомментирована или укажите путь к database/database.sqlite

Создайте файл БД и выполните миграции:

bash
touch database/database.sqlite
php artisan migrate
Сгенерируйте ключ приложения:

bash
php artisan key:generate
Запустите встроенный сервер:

bash
php artisan serve
✅ Проверка заданий
###  1. Консольная команда и планировщик
   Ручной запуск:
   php artisan jokes:fetch – будет получена и сохранена новая шутка.

Автоматический каждые 5 минут:
php artisan schedule:work (оставьте терминал открытым – команда будет выполняться каждые 5 минут).

Проверка сохранения:
php artisan tinker → App\Models\Joke::all();

### 2. API /api/jokes
   Запустите сервер (php artisan serve).

Откройте в браузере или выполните curl:
http://127.0.0.1:8000/api/jokes
Должен вернуться JSON-массив со всеми шутками.

### 3. JavaScript + динамические поля (страница testlist.html)
   Файл public/js/dynamic-fields.js содержит готовый скрипт.

Как протестировать:

Откройте страницу http://test.amopoint-dev.ru/testzz/testlist.html (или локальную копию).

Откройте консоль браузера (F12) и вставьте содержимое dynamic-fields.js.

При выборе значения в выпадающем списке будут отображаться только поля, чей числовой суффикс совпадает с выбранным числом (например, input_1, button_1 при выборе 1). Скрытые поля отключаются (disabled), их значения не отправляются.

#### Обоснование выбора алгоритма (из задания):

Алгоритм	Почему не выбран
Поиск подстроки includes()	Показывал бы button_12 при выборе 1 – неверно.

Точное равенство имени name === 'input_1'	Игнорирует кнопки и другие элементы, не гибко.

Использование jQuery	Страница уже подключает jQuery, но чистый JS уменьшает зависимости и демонстрирует знание нативного API.

Регулярное выражение _(\d+)$	Эквивалентно split('_') и parseInt(), но менее читаемо.

Скрытие без disabled	Значения скрытых полей всё равно уходят на сервер – приводит к ошибкам валидации.

Добавление data- атрибутов	Требует изменения HTML, что запрещено условием.

Выбранный алгоритм:

Извлечение числового суффикса после последнего подчёркивания и сравнение с выбранным числом.

Точное соответствие (исключает ложные срабатывания button_12).

Работает с любыми элементами (input, button, select и т.д.).

Отключает скрытые поля (disabled=true).

Не требует изменения исходной разметки.
